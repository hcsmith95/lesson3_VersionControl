Lesson 3 Highlights

Harrison Smith

Highlight #1 Version Control Systems

A version control system is a a system that allows web developers to save any and all files that they might be working on to a remote repository, so there's no chance of losing one's work if one has a computer malfunction. Version control allows developers to access every version of a document, so even after it's updated, it's easy to revert back to a previous copy if need be. Because everything (theoretically) is saved to an external site, there's no chance of losing your data, or other files. It also allows users to share files with one another easily, without having to copy and paste and send documents to each other every time they get updated. This way, everyone who needs access to the files has immediate access to any and all versions of each and every file.

Highlight #2 Github

Github is an online version control remote repository and web hosting service. Users can back up any and all files to Github repositories. These files can be saved and shared with anyone else with a Github account. Anyone with access to such files, also has access to every single version of each file, so, if need be, documents or code from a pervious version can be retrieved and used again, in case someone makes a mistake, and is unable to fix said mistake. This ensures that there's no "punishment" for experimenting and trying new things in coding, because you can always go back to a past version.

Highlight #3 GIT Commands

There are a few very basic GIT commands, that everyone needs to know and understand. "Git Add" sends all of the selected files to a staging area, where the user has a chance to double check which files they do and don't want to save. "Git Commit" "commits" all of the chosen files and places them into a local repository, where they are stored and saved. "Git Push" pushes all of the files that are in the local repository to the remote repository, which is essentially another spot to store and save files, adding another security blanket so you know you'll never lose your work.

Highlight #4 GIT Repository

The Git repository is a digital storage area where users with a Github account can store all of their documents and files, to ensure there's no chance of losing any files. The GIT repository also allows users to share the files that they have saved to their repositories with anyone they choose to, so long as that other person also has a Github account. The Git repository also stores every version of each document or file. Because of this, anyone can revert back to a previous version of a document, so if someone is experimenting with a new idea, and they try it out, and later decide they prefer another version instead, it's not a problem to retrieve that older version.

Highlight #5 Local vs Remote Repositories

A local repository is located on whatever platform you're developing on. In my case, that local repository can be found on Cloud9 servers. However, many developers want an added layer of security around their code, so a remote repository is a good place to store them. In the event that Cloud9 servers ever fail, a remote repository like Github also has all the files saved on their servers. Because of this, there's virtually no chance of losing one's work because of a computer malfunction. It's always nice to be completely sure there's no chance of losing any progress you've made, so you have one less thing to lose sleep over.
